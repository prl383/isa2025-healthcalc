Especificación de caso de uso.
1.Nombre
	Estimar calorías diarias necesarias.
2.Actor Principal
	Usuario (usuario que quiere saber cuantas calorías debe consumir diariamente siendo activo)
3.Ámbito
	Calculadora HealthCalc.
4.Nivel
	Alto nivel, relacionado con la funcionalidad principal de la aplicación.
5.Stakeholders y sus intereses
	Usuario: Desea conocer cuantas calorías diarias debe consumir segun su actividad física.
	Desarrolladores: Quieren que la aplicación funcione adecuadamente.
6.Precondiciones
	El usuario debe insertar los siguientes parámetros para realizar el cáculo:
	-Género
	-Altura(cm)
	-Peso(kg)
	-Actividad física
	-Edad
7.Garantías mínimas
	Se muestra un mensaje de error si los parámetros ingresados no son válidos o falta algun parámetros.
8.Garantías de éxito	
	Se muestra al usuario las calorías necesarias según los datos aportados.
9.Escenario principal
	9.1. El usuario selecciona la opción "Estimar calorías diarias necesarias" de la calculadora.
	9.2. El sistema solicita los datos al usuario (Altura, peso, edad, genero, actividad).
	9.3. El usuario introduce los datos y confirma la operación.
	9.4. El sistema calcula la TMB según los datos introducidos por el usuario y los muestra por pantalla.
	9.5. El usuario recibe los resultados y puede realizar otro cálculo.
	
10.Escenarios alternativos
	10.3.1 El usuario no ingresa todos los datos.
		-El sistema muestra un mensaje indicando que faltan campos a rellenar.
	10.3.2	El usuario ingresa datos erroneos
		-El sistema muestra un mensaje de error indicando que los datos son incorrectos.
